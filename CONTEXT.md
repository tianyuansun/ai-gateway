# Domain Glossary — AI Gateway

- **Gateway** — the reverse proxy that receives coding-agent API requests, translates them between formats, and routes them to upstream providers.
- **Exposed format** — the API format the gateway presents to clients: Responses API, Chat Completions API, or Anthropic Messages API.
- **Provider** — an upstream LLM service (e.g. DeepSeek, OpenRouter, Anthropic). Each provider exposes one or more endpoints in a *native format*.
- **Native format** — the API format a provider accepts natively (Chat or Anthropic). The gateway translates the exposed format to the native format when they differ.
- **Translator** — a component that converts requests and responses between an exposed format and a native format. Includes a *PassthroughTranslator* that returns data unchanged when the two formats match.
- **Session** — holds minimal cross-turn state for provider affinity: `ID`, `ProviderID`, `ModelName`, and lifecycle fields (`CreatedAt`, `LastAccess`, `TTL`). Does NOT store message history or reasoning records — every coding agent (Codex CLI, Claude Code, Aider, Cursor, Copilot, etc.) sends the full conversation history in each request body. Reasoning content flows through the request body as well (Codex CLI sends `ResponseItem::Reasoning` in `input`). The `session.Store` interface (Get/Set/Delete/Prune) supports pluggable backends: `MemoryStore` (in-process LRU), `SQLiteStore` (single-file durable, for surviving restarts), and `RedisStore` (future, for distributed deployments). See `docs/adr/0006-session-store-stateless.md`.
- **Request log buffer** — a per-request ring buffer of structured log records, carried via `context.Context`. Drained (flushed to output) when a *trigger condition* fires; otherwise discarded.
- **Trigger condition** — a rule that causes a request's buffered logs to be flushed: error response, upstream failure, latency exceeding a threshold, client cancellation, or explicit debug header (`X-Debug: true`).
- **Log level** — controls the minimum severity that enters the buffer: `debug`, `info`, `warn`, `error`. Global default + per-request override via `X-Debug` header.
- **Schema types** — Go structs that model the request/response shapes of each external API format (Responses API, Chat Completions API, Anthropic Messages API). Generated from upstream OpenAPI/JSON Schema specs; not hand-written. Three independent type sets — one per API format — with translators operating directly between them.
- **Model catalog** — the list of available models exposed at `GET /v1/models`. Returns either OpenAI format (`{object: "list", data: [...]}` with flat capabilities) or Anthropic format (`{data: [...], has_more, ...}` with nested `{capability: {supported: bool}}` capabilities), detected from the request's auth header or `Accept` content type.
- **Compact** — Codex CLI's context window compaction, which has three strategies: (1) *local compact* — `POST /v1/responses` (streaming) with a `SUMMARIZATION_PROMPT` in the `instructions` field; used for all non-OpenAI providers, including the gateway; (2) *remote compact v1* — `POST /v1/responses/compact` (unary); used for OpenAI/Azure when the v2 feature flag is disabled; (3) *remote compact v2* — `POST /v1/responses` (streaming) with a `CompactionTrigger` input item and `Compaction` output; used for OpenAI/Azure when the v2 feature flag is enabled. The gateway only receives local compact requests. The existing `handleProxy` → `ResToChat` pipeline handles them, provided `instructions` is forwarded as a system message. See `docs/adr/0005-compact-translator-pipeline.md`.
- **Compact model** — an optional per-model configuration (`compact_model`) that specifies a different (typically cheaper) model to use for compaction summarization. Deferred.
